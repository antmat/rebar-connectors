# Параметры посадки винтовой PETG-втулки — проект

Дата: 2026-07-15

## Контекст

Физическая примерка показала, что вариант `wide` навинчивается приемлемо, но
параметры модели не соответствуют тому, чем фактически управляется посадка:

- наплывы на торце арматуры мешают пройти через жёсткое кольцо буртика;
- рабочая часть втулки состоит из двух гибких винтовых лент, поэтому в разумном
  диапазоне её посадку определяет прежде всего толщина лент, а не единый наружный
  диаметр;
- ширина винтовой прорези после успешной примерки больше не должна быть вариантом
  `Fit`.

Измеренный диаметр сердцевины арматуры без навивки — около 9,1 мм. Максимальный
диаметр по двум навивкам — около 11,3 мм. Ширина навивки — 2,5 мм, ход каждого из
двух заходов — 45 мм.

## Цель

Сделать основные параметры Customizer физически понятными:

1. измеренный диаметр сердцевины;
2. диаметральный зазор к сердцевине;
3. базовая толщина стенки винтовой ленты.

`Fit` должен выбирать небольшую поправку к толщине стенки. Ширина прорези и
дополнительное раскрытие входа в буртике остаются отдельными вторичными параметрами.

## Основные параметры и производные размеры

Базовая конфигурация:

| Параметр | Значение | Смысл |
|---|---:|---|
| `Core_D_mm` | 9,1 мм | измеренный диаметр без навивки |
| `Core_Diametral_Clearance_mm` | 0,7 мм | общий зазор по диаметру, не на сторону |
| `Wall_Thickness_mm` | 1,3 мм | толщина ленты для `medium` |

Рабочий диаметр отверстия вычисляется как
`Core_D_mm + Core_Diametral_Clearance_mm = 9,8 мм`.

Варианты `Fit` меняют только толщину ленты:

| `Fit` | Толщина ленты | Производный наружный диаметр |
|---|---:|---:|
| `loose` | 1,2 мм | 12,2 мм |
| `medium` | 1,3 мм | 12,4 мм |
| `tight` | 1,4 мм | 12,6 мм |

Шаг между вариантами — 0,1 мм радиальной толщины. Формулы:

```text
bore_d = Core_D_mm + Core_Diametral_Clearance_mm
wall_t = Wall_Thickness_mm + fit_offset
cage_d = bore_d + 2 * wall_t
```

Названия `loose` и `tight` описывают ожидаемую посадку, но окончательный выбор
делается только физической примеркой. Наружный диаметр может немного превышать
номинальный диаметр гнезда: две раздельные винтовые ленты пружинят внутрь. Поэтому
между этими двумя размерами не должно быть запрещающего `assert()`.

## Вторичные параметры

- `Rib_Slot_Width_mm = 3,5 мм` — одинаковая ширина винтовой прорези для всех
  вариантов `Fit`; она больше ширины навивки 2,5 мм и даёт по 0,5 мм бокового
  запаса на сторону.
- `Entry_Extra_Clearance_mm = 1,0 мм` — дополнительное диаметральное раскрытие
  отверстия только на наружной плоскости буртика.
- `Socket_D_mm = 12,4 мм` — справочный размер напечатанного гнезда и размер для
  `assembly_preview`; он не управляет геометрией втулки.
- `Rebar_Max_D_mm = 11,3 мм` и `Rib_Radial_Clearance_mm = 0,3 мм` сохраняют
  назначение для глубины винтовых каналов.

Вход в буртике выполняется коротким коническим переходом: на наружной плоскости
диаметр равен `bore_d + Entry_Extra_Clearance_mm = 10,8 мм`, а у внутренней
плоскости буртика сужается до рабочего диаметра 9,8 мм. Выше буртика отверстие
остаётся цилиндрическим диаметром 9,8 мм. Такой вход пропускает торцевые наплывы,
не ослабляя всю рабочую длину.

## OpenSCAD-интерфейс

Режимы сохраняются: `calibration_set`, `calibration_single`, `full` и
`assembly_preview`. По умолчанию показывается полноразмерная деталь `full` с
`Fit = "medium"`.

Внутренние функции вычисляют рабочий диаметр отверстия, толщину выбранного
варианта, наружный диаметр и входной диаметр. Модуль детали получает вариант
`Fit`, а не готовую ширину прорези. Сквозные винтовые прорези используют один
`Rib_Slot_Width_mm = 3,5 мм` во всех вариантах.

Калибровочный набор содержит три детали `loose`, `medium`, `tight`. Их можно
различить по одной, двум и трём меткам на буртике. Полноразмерный стандартный STL
экспортируется для `medium`.

Диагностический вывод должен явно показывать минимум: измеренный диаметр
сердцевины, диаметральный зазор, рабочий диаметр отверстия, выбранную толщину
стенки, производный наружный диаметр, ширину прорези и входной диаметр буртика.

Проверки параметров должны отклонять неположительные размеры, неизвестный `Fit`,
толщину не больше нуля, рабочее отверстие не меньше диаметра клетки, ширину
прорези не больше ширины навивки, входной диаметр меньше рабочего либо не меньше
наружного диаметра буртика. Проверка `cage_d < Socket_D_mm` удаляется.

## Артефакты и документация

Экспорт создаёт:

- `helical_insert_calibration_loose.stl`;
- `helical_insert_calibration_medium.stl`;
- `helical_insert_calibration_tight.stl`;
- общий калибровочный набор, полноразмерный `medium` и существующие PNG-превью.

Старые отдельные файлы `narrow` и `wide` удаляются как вводящие в заблуждение.
Инструкция по печати объясняет, что сначала выбирается толщина стенки, а ширина
прорези 3,5 мм меняется только при отдельной необходимости.

## Проверка

Автоматическая проверка подтверждает формулы размеров, одинаковую прорезь 3,5 мм
у трёх вариантов, конический вход 10,8→9,8 мм, правильную правую спираль, число
связных компонентов и замкнутость STL без non-manifold рёбер.

Финальное доказательство посадки остаётся физическим: торец с наплывами должен
проходить буртик, втулка должна навинчиваться рукой по часовой стрелке и вместе с
арматурой входить в реальное гнездо без разрушения.
