# Крепления для пластиковой арматуры

Основной файл модели — `rebar_connectors.scad`. Он создаёт три монолитных
крепления для восьмиугольной конструкции: нижний трёхлучевой угол, верхний
четырёхлучевой угол и восьмилучевую верхушку. Поддержки в геометрию не
встроены: их добавляет слайсер.

## Параметры по умолчанию

| Параметр Customizer | Значение | Назначение |
|---|---:|---|
| `Hole_D_mm` | 12,2 мм | Номинальный диаметр канала |
| `Socket_Depth_mm` | 30 мм | Глубина посадки от устья до глухого упора |
| `Socket_Wall_mm` | 4 мм | Радиальная толщина стенки; наружный диаметр 20,2 мм |
| `Entry_Chamfer_mm` | 1 мм | Номинальная длина входной фаски около 45° |
| `Min_Web_mm` | 3 мм | Минимальная перемычка между соседними каналами |
| `Roof_Angle_deg` | 30° | Угол лучей крыши к горизонтали |

Глубина посадки зафиксирована на 30 мм: для этого размера проверены глухой
упор, перемычки и раскладка `print_set` на столе 220 × 220 мм.

Диаметр 12,2 мм — геометрический размер модели. Перед большой печатью
напечатайте `fit_test.stl`. Если отверстия конкретного принтера получаются
меньше, скорректируйте `Hole_D_mm`, а не масштаб всей детали.
`Min_Web_mm=3` задаёт нижнюю границу, поэтому фактическая перемычка может быть
немного толще. Фаска имеет небольшой технологический нахлёст булевых тел и не
является измерительным эталоном точного угла 45°.

## Режимы Customizer

Параметр `Part` выбирает вывод:

- `lower_3way` — нижний угол: два горизонтальных порта под внутренним углом
  135° и один вертикальный порт вверх;
- `upper_4way` — верхний угол: два горизонтальных порта, вертикальный порт
  вниз и луч крыши под 30°;
- `apex_8way` — верхушка с восемью портами через 45° по азимуту;
- `print_set` — три разные детали, уже разложенные для печати;
- `fit_test` — один короткий калибровочный стакан с теми же диаметром канала,
  глубиной и входной фаской;
- `assembly_preview` — условная сборка из двух восьмиугольников, восьми
  вертикалей и восьми лучей крыши; этот режим не предназначен для STL.

`Section_View=true` разрезает выбранный вывод плоскостью X=0, оставляет
половину геометрии и открывает центральный разрез каналов. Это диагностический
вид, а не печатная модель;
для `assembly_preview` разрез намеренно не применяется. `Diagnostics=true`
выводит рассчитанные размеры в консоль OpenSCAD. Параметры
`Preview_Side_mm=300` и `Preview_Height_mm=500` влияют только на условную
сборку.

В интерактивном режиме отдельные узлы сохраняют ориентацию сборки. Готовые
индивидуальные STL, созданные скриптом ниже, уже повёрнуты в рекомендованную
ориентацию печати и лежат на плоскости Z=0. `print_set` также уже ориентирован.

## Автоматический экспорт

Нужен OpenSCAD с backend Manifold. В комплекте проекта уже находится BOSL2.
На macOS скрипт по умолчанию использует установленное приложение OpenSCAD и
на Apple Silicon запускает его через `arch -x86_64`:

```bash
scripts/render_rebar_connectors.sh
```

Другой исполняемый файл можно указать через `OPENSCAD_BIN`:

```bash
OPENSCAD_BIN=/путь/к/OpenSCAD scripts/render_rebar_connectors.sh
```

Каждый экспорт выполняется с `--backend Manifold --hardwarnings`. Результаты
появляются в `build/`:

| Файл | Назначение |
|---|---|
| `lower_3way.stl` | Одна нижняя угловая деталь в печатной ориентации |
| `upper_4way.stl` | Одна верхняя угловая деталь в печатной ориентации |
| `apex_8way.stl` | Одна центральная верхушка в печатной ориентации |
| `fit_test.stl` | Проверка посадки прутка; закрытое дно уже лежит на столе |
| `print_set.stl` | По одной детали каждого типа на одном столе |
| `assembly_preview.png` | Контроль сборки в перспективе |
| `assembly_preview_side.png` | Контроль высот и угла крыши сбоку |

Для полной восьмиугольной конструкции нужны 8 × `lower_3way.stl`,
8 × `upper_4way.stl` и 1 × `apex_8way.stl`; `print_set.stl` содержит только
по одному экземпляру каждого типа.

Фактический габарит `print_set.stl` по XY — **169.643 × 205.510 мм**. Нужна
рабочая область стола не менее **220 × 220 мм**. Проверьте, что край стола,
клипсы и выбранная ширина каймы не уменьшают доступную область.

## Начальные настройки печати

- материал деталей — PETG;
- материал внутренних поддержек или их контактного интерфейса — PLA при
  наличии двухматериальной печати;
- слой — 0,20 мм;
- не менее 5 периметров;
- заполнение — 40–60%;
- не менее 5 верхних и 5 нижних слоёв;
- поддержки — везде, включая внутренние своды каналов; генерировать слайсером;
- кайма для удержания детали — ориентировочно 5–8 мм, если она помещается на
  выбранном столе;
- температуры и обдув — по рекомендациям производителя конкретного PETG и
  PLA.

Сначала напечатайте `fit_test.stl`, удалите поддержку и проверьте посадку
реального прутка. Затем печатайте индивидуальные STL в нужном количестве или
`print_set.stl` для пробного набора из трёх разных деталей.

## TPU-вкладыши для арматуры 11,3 мм

Файл `rebar_insert.scad` создаёт разрезной TPU-вкладыш для установки арматуры
диаметром 11,3 мм в уже напечатанные каналы диаметром 12,2 мм. Вкладыш имеет
втулку длиной 26 мм, стенку 0,45 мм, продольный разрез 1 мм и удерживающий
буртик диаметром 15 мм.

Сначала напечатайте полный калибровочный набор `tpu_insert_fit_set.stl` и
проверьте каждый вариант с реальной арматурой в реальном креплении:

| Метка на буртике | Посадка | Внутренний диаметр | Наружный диаметр |
|---|---|---:|---:|
| 1 выступ | свободная (`loose`) | 11,30 мм | 12,20 мм |
| 2 выступа | средняя (`medium`) | 11,20 мм | 12,10 мм |
| 3 выступа | плотная (`tight`) | 11,10 мм | 12,00 мм |

Рекомендуемые начальные настройки Bambu Studio:

- материал — [Bambu TPU for AMS 68D](https://us.store.bambulab.com/products/tpu-for-ams),
  предварительно высушенный;
- сопло — 0,4 мм;
- слой — 0,20 мм;
- генератор стенок — Arachne;
- поддержки — выключены;
- ориентация — буртиком на столе, втулкой вертикально.

Перед печатью обязательно проверьте предпросмотр слайсера по всей высоте:
стенка 0,45 мм должна формироваться одной непрерывной линией без пропусков.
Сквозной канал вкладыша не меняет глубину посадки, поэтому длины из
`scripts/rebar_cut_list.py` остаются прежними.

Готовые файлы экспортируются отдельно от основных креплений:

```bash
scripts/render_rebar_insert.sh
```

Скрипт создаёт в `build/` файлы `tpu_insert_loose.stl`,
`tpu_insert_medium.stl`, `tpu_insert_tight.stl`,
`tpu_insert_fit_set.stl` и `tpu_insert_fit_set.png`.

## Расчёт длины арматуры

Исполняемый Python-скрипт строит ведомость раскроя по диаметру и высоте:

```bash
scripts/rebar_cut_list.py --diameter 3000 --height 2000
```

`--diameter` — расстояние между центрами противоположных вершин восьмиугольника,
а `--height` — расстояние между центрами нижних и верхних креплений. Все исходные
размеры задаются в миллиметрах.

Результат учитывает глухие упоры текущей модели креплений и выводит точные длины
реза без запаса на отходы или ошибки. Общий метраж рассчитывается по неокруглённым
значениям, поэтому может на 0,001–0,002 м отличаться от суммы показанных округлённых строк.
Для закупки материала запас нужно добавить отдельно.
