# Tagging — тэгирование и чистые булевы операции

> Раздел: [BOSL2 — справочник](../BOSL2.md) › ключевые приёмы.
> Источник: `attachments.scad`.

Тэги заменяют громоздкие `difference()`/`intersection()` с правильной иерархией детей:
операция определяется не вложенностью, а **тегами** на деталях. Это делает вычитание
устойчивым к параметризации (не «ломается» при изменении размеров).

---

## 1. Назначение тэгов

| Модуль | Назначение |
|--------|-----------|
| `tag("имя") CHILDREN;` | присвоить тег всем детям. |
| `tag_this("имя") CHILDREN;` | тег только на текущем уровне (не распространяется глубже). |
| `force_tag("имя") CHILDREN;` | тег для **не-attachable** объектов (которые игнорируют обычный `tag`). |
| `default_tag("имя", [do_tag]) CHILDREN;` | тег по умолчанию, если у ребёнка своего нет. |
| `tag_scope([scope]) CHILDREN;` | новый контекст тегов (изолирует область). |

**Стандартные теги операций:** `"remove"` (вычесть), `"keep"` (сохранить), `"intersect"` (пересечение).

---

## 2. Операции с тегами

### diff([remove="remove"], [keep="keep"]) PARENT() CHILDREN;
Из родителя вычитается всё с тегом `remove`; с тегом `keep` — сохраняется принудительно.
```openscad
diff()
  cuboid([40,30,10]) {
    tag("remove") attach(TOP,BOT,inside=true) cyl(h=20, d=8);   // отверстие
    tag("remove") align(FRONT,TOP,inside=true,shiftout=0.01) cuboid([20,5,4]); // паз
  }
```
`tag_diff([tag], [remove], [keep])` — то же + ставит тег на результат.

### intersect([intersect="intersect"], [keep]) PARENT() CHILDREN;
Оставить только пересечение родителя с детьми-тегом `intersect`.
`tag_intersect(...)` — + тег на результат.

### conv_hull([keep]) CHILDREN;  /  tag_conv_hull([tag], [keep])
Выпуклая оболочка детей (с тегом `keep` можно исключить часть).

> В `diff`/`intersect` родитель — **первый** ребёнок (обычно основное тело), а вычитаемые/пересекаемые
> объекты помечаются `tag("remove")`/`tag("intersect")` и тоже лежат внутри.

---

## 3. Сокрытие/показ

| Модуль | Действие |
|--------|----------|
| `hide(tags)` | скрыть детей с указанными тегами. |
| `hide_this()` | скрыть детей на текущем уровне. |
| `show_only(tags)` | показать **только** детей с этими тегами. |
| `show_all()` | показать всё, очистить теги. |
| `show_int(tags)` | показать детей с тегами, уже видимых в родительском контексте. |

Удобно для отладки сборок: помечайте отладочные детали `tag("debug")` и `hide("debug")`.

---

## 4. Паттерны и правила

1. **Предпочитайте `diff() {...}` ручному `difference()`** — читается плоско, не зависит от иерархии.
2. Отверстия/выемки ставьте через `attach(...,inside=true)` или `align(...,inside=true)` —
   они автоматически получают тег `"remove"`, поэтому можно писать `diff() PARENT() attach(...) CHILD;`.
3. Используйте `$slop` / `overlap` / `shiftout=0.01` против Z-fighting (см. [CONVENTIONS §5](../CONVENTIONS.md)).
4. Для многослойной сборки оборачивайте группы в `tag_scope()`, чтобы теги не «протекали» наружу.

```openscad
diff("remove","keep")
  cuboid([50,40,15]) {
    tag("remove") xcopies(20) attach(TOP,BOT,inside=true) cyl(h=30, d=3);  // 2 отверстия
    tag("keep")    position(TOP) cuboid([50,40,2]);                       // крышка-ребро
  }
```
