# Attachments — анкеринг и относительное позиционирование

> Раздел: [BOSL2 — справочник](../BOSL2.md) › ключевые приёмы.
> Источник: `attachments.scad`. **Самый важный раздел** для понимания BOSL2.

Система attachments позволяет ставить дочерние объекты относительно родителя (граней, рёбер, углов),
а также задавать позицию/ориентацию через `anchor`/`spin`/`orient`. Работает только если и родитель,
и ребёнок написаны с поддержкой attachable (все примитивы BOSL2 — поддерживают).

---

## 1. Три параметра позиционирования: `anchor` / `spin` / `orient`

Порядок применения: **anchor → spin → orient**.

### anchor (анкер)
`anchor=` транслирует объект так, что **анкерная точка попадает в начало координат** (0,0,0).
Анкер — это вектор направления (или имя). Для прямоугольных объектов компоненты — `0` или `±1`:

* грань (face) = одно направление: `TOP`=[0,0,1], `BOTTOM`, `LEFT`, `RIGHT`, `FRONT`, `BACK`;
* ребро (edge) = сумма двух: `TOP+RIGHT`=[1,0,1] → центр верхне-правого ребра;
* угол (corner) = сумма трёх: `TOP+RIGHT+BACK`.

Для криволинейных объектов (цилиндр, сфера) анкер можно задать произвольным вектором — точка
берётся на поверхности (пересечением), направление = нормаль поверхности.

**В 2D:** `TOP`/`BOTTOM` интерпретируются по оси Y (если y-компонента анкера = 0, то Z→Y).

### spin
`spin=` — поворот **против часовой стрелки вокруг Z** (в градусах). Применяется **после** anchor,
поэтому вращает не вокруг центра объекта — учитывайте это.

### orient
`orient=` — поворот объекта так, что его верх (TOP) смотрит в заданном направлении.
Применяется последним. Напр. `orient=RIGHT` кладёт объект на бок.

> `anchor`/`spin`/`orient` **игнорируются**, когда ребёнок ставится через `align()` или
> двухаргументный `attach()` — они задают свою позицию/ориентацию.

## 2. Константы направлений (`constants.scad`)

| Константа | Значение | Алиасы |
|-----------|----------|--------|
| `RIGHT` | `[1,0,0]` | — |
| `LEFT`  | `[-1,0,0]` | — |
| `BACK`  | `[0,1,0]` | — |
| `FRONT` | `[0,-1,0]` | `FWD`, `FORWARD` |
| `UP`    | `[0,0,1]` | `TOP` |
| `DOWN`  | `[0,0,-1]` | `BOTTOM`, `BOT` |
| `CENTER` | `[0,0,0]` | `CTR`, `CENTRE` |

---

## 3. Дефолтный анкер — ИСПРАВЛЕНИЕ частой ошибки

**Большинство примитивов BOSL2 по умолчанию `anchor=CENTER`.** Исключения:

| Примитив | anchor по умолчанию |
|----------|---------------------|
| `cube`, `cuboid`, `cyl`, `xcyl/ycyl/zcyl`, `tube`, `spheroid`, `torus`, `teardrop`, `onion`, `regular_prism`, `pie_slice` | **`CENTER`** |
| `prismoid` | **`BOTTOM`** |
| `cylinder` (нативная обёртка) | **`BOTTOM`** |
| 2D-примитивы (`rect`, `circle`, …) | **`CENTER`** |

> Не предполагайте, что 3D-объект «стоит на плоскости» — большинство центрировано.
> Чтобы поставить на основание: `anchor=BOT` или `center=true` (где поддерживается).

## 4. Спецификация наборов: грани, рёбра, углы

Используется в `edges=`/`except=` примитивов и в модулях масок `edge_profile`/`corner_mask`/...

### Грани
Любой вектор направления → одна грань: `TOP`, `RIGHT`, `BACK`...

### Наборы рёбер (`edges=` / `except=`)
Дескриптор ребра — один из:
* **вектор на ребро** → это ребро (`TOP+FRONT`);
* **вектор на грань** → все рёбра вокруг грани (`TOP`);
* **вектор на угол** → все рёбра, сходящиеся в углу (`TOP+RIGHT+BACK`);
* строка `"X"`/`"Y"`/`"Z"` — все рёбра вдоль этой оси;
* строка `"ALL"` (по умолчанию) / `"NONE"`;
* 3×4 массив флагов (12 рёбер).

`edges=` — включить, `except=` — исключить. Можно списком: `edges=[TOP,"Z"]`.
См. [cuboid.md](./cuboid.md) для примеров.

### Наборы углов (`corners=` / `except=`)
Аналогично: вектор на угол / на ребро (оба конца) / на грань (4 угла грани), `"ALL"`, `"NONE"`,
или 8-вектор флагов в порядке `[X-Y-Z-, X+Y-Z-, X-Y+Z-, X+Y+Z-, X-Y-Z+, X+Y-Z+, X-Y+Z+, X+Y+Z+]`.

### Сервисные функции
`EDGE(i)` — анкер ребра по индексу; `EDGE(TOP,i)`/`EDGE(BOT,i)` — верхнее/нижнее ребро;
`FACE(i)` — анкер грани. Используются для N-гранных призм и цилиндров.

## 5. Тип анкера (`atype`)

Для непрямоугольных объектов доступно несколько схем анкерования (через `atype=`):

| atype | Поведение |
|-------|-----------|
| `"box"` | (по умолч.) анкеры как у «огибающего» прямоугольника, без учёта скруглений. Удобно для позиционирования «как куба». |
| `"perim"` | анкеры на реальном периметре (по кромке скругления). |
| `"intersect"` | точка пересечения вектора анкера с поверхностью (для цилиндров/сфер — по умолчанию). |
| `"hull"` | точка на выпуклой оболочке; для вогнутых форм не «проваливается» внутрь. |

## 6. Модули позиционирования (после родителя)

Форма вызова: `PARENT() module(...) CHILDREN;`

### position(at) — поставить без поворота
Дочерние объекты переносятся в точку-анкер `at` родителя; **ориентация детей не меняется**.
Нужно вручную задавать `anchor=` детям.
```openscad
cuboid([40,30,20]) position(TOP) cyl(h=5, d=10, anchor=BOT);
```

### orient(anchor, [spin]) — повернуть детей к анкеру
Наклоняет детей так, чтобы их верх смотрел в сторону `anchor` родителя.
```openscad
prismoid([50,50],[30,30],h=40) position(TOP+RIGHT) orient(RIGHT) prismoid([30,30],[0,5],h=20);
```

### align(anchor, [align], [inside], [inset], [shiftout], [overlap]) — прижать к кромке
Ставит ребёнка на грань `anchor`, выравнивая его по направлению `align` (ребро/угол).
**Сам подбирает нужный `anchor` ребёнка** → `anchor=`, заданный ребёнку, **игнорируется**.
Список анкеров/направлений → `align()` работает как дистрибьютор (множество копий).
* `inside=true` → ребёнок внутри родителя (автоматический тег `"remove"`, удобно для `diff()`).
* `shiftout=0.01` — убрать Z-fighting; `overlap=` — утопить/выдвинуть; `inset=` — отступить от кромки.

### attach(parent, child, ...) — позиция + ориентация ⭐ основной инструмент
Двухаргументная форма: анкер `child` ребёнка стыкуется к анкеру `parent` родителя,
ребёнок **разворачивается по нормали** грани. Параметры: `align=`, `spin=`, `overlap=`, `inside=`,
`inset=`, `shiftout=`.
Одноаргументная форма `attach(parent)` — дети «растут» из грани `parent` наружу (как продолжение).
```openscad
cuboid([40,30,10]) attach(TOP,BOT) cyl(h=20, d=8);   // цилиндр сверху, снизу анкер BOT
diff() cuboid([40,30,10])
  attach(FRONT,TOP,inside=true,shiftout=0.01) cuboid([10,5,4]);  // выемка-паз
```

> **`position`** = только перенос. **`align`** = перенос + авто-подбор анкера ребёнка (без вращения).
> **`attach`** = перенос + ориентация по нормали грани.

## 7. Свои attachable-объекты

* `attachable(anchor, spin, orient, size=|r=|path=|vnf=|geom=, ...) { OBJECT; children(); }` —
  обёртка, превращающая произвольный модуль в attachable. Вызовите `children()` внутри для поддержки вложенных attach-операций.
* `reorient(...)` — функция-вариант: возвращает матрицу/точки для ручного анкерования.
* `named_anchor(name, pos, [orient], [spin])` — определить именованный анкер.
* `attach_geom(...)` / `define_part(...)` — описание геометрии / именованной части.
